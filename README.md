## Описание проекта

no-code-ui — это десктопное приложение для визуального редактирования React, React Native и HTML файлов. Приложение позволяет:

- 🎨 **Просматривать и редактировать файлы в визуальном режиме** — интерактивный блочный редактор
- 💻 **Редактировать код в текстовом редакторе** — встроенный Monaco Editor (VS Code)
- 🧱 **Работать с блочной структурой элементов** — визуальная иерархия компонентов
- 📁 **Управлять проектом через файловое дерево** — создание, удаление, переименование файлов
- ⚡ **Отслеживать изменения в реальном времени** — автообновление при изменении файлов
- ⌨️ **Использовать горячие клавиши** — Ctrl+S для сохранения, Ctrl+Z/Ctrl+Shift+Z для отмены/повтора
- 📜 **Управлять историей изменений** — полная поддержка Undo/Redo операций
- 🎯 **Накапливать изменения перед сохранением** — система staged patches

## Связь фронтенда и бэкенда

### IPC (Inter-Process Communication) архитектура

Связь между фронтендом и бэкендом осуществляется через Electron IPC с использованием безопасного моста через `preload.js`.

#### 1. Preload Script (electron/preload.js)

Preload скрипт создает безопасный мост между процессами, используя `contextBridge`:

```javascript
// electron/preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('electronAPI', {
  readFile: (filePath) => ipcRenderer.invoke('fs:readFile', filePath),
  writeFile: (filePath, content, options) => 
    ipcRenderer.invoke('fs:writeFile', filePath, content, options),
  // ... другие методы
});
```

**Безопасность:**
- `nodeIntegration: false` — отключена интеграция Node.js в рендер-процесс
- `contextIsolation: true` — изоляция контекста для безопасности
- Все API доступны только через `window.electronAPI`

#### 2. Main Process (electron/main.js)

Главный процесс обрабатывает IPC запросы через `ipcMain.handle`:

```javascript
// electron/main.js
ipcMain.handle('fs:readFile', async (event, filePath) => {
  try {
    const content = await fs.readFile(filePath, 'utf-8');
    return { success: true, content };
  } catch (error) {
    return { success: false, error: error.message };
  }
});
```

#### 3. Frontend API (src/shared/api/electron-api.js)

Фронтенд использует абстракцию для работы с Electron API:

```javascript
// src/shared/api/electron-api.js
export async function readFile(filePath) {
  if (!window.electronAPI) {
    return { success: false, error: 'Electron API не доступен' };
  }
  return await window.electronAPI.readFile(filePath);
}
```

### Основные IPC каналы

#### Файловые операции
- `fs:readFile` — чтение файла
- `fs:writeFile` — запись файла
- `fs:readDirectory` — чтение директории
- `fs:readFileBase64` — чтение файла в base64
- `fs:deleteFile` — удаление файла
- `fs:deleteDirectory` — удаление директории
- `fs:rename` — переименование файла/директории
- `fs:ensureDir` — создание директории

#### Диалоги
- `dialog:openFile` — диалог выбора файла
- `dialog:openDirectory` — диалог выбора директории

#### Отслеживание файлов
- `fs:watchFile` — начало отслеживания файла
- `fs:unwatchFile` — остановка отслеживания
- `file:changed` — событие изменения файла (от Main к Renderer)

### Пример использования

```javascript
// В React компоненте
import { readFile, writeFile } from './shared/api/electron-api';

// Чтение файла
const result = await readFile('/path/to/file.jsx');
if (result.success) {
  console.log(result.content);
}

// Запись файла
const writeResult = await writeFile('/path/to/file.jsx', newContent, {
  backup: true // создаст .bak файл
});
```

## Основные компоненты

### 1. RenderFile.jsx

Главный компонент для рендеринга и редактирования файлов. Поддерживает:
- HTML файлы
- React компоненты (JSX/TSX)
- React Native компоненты
- Обработку зависимостей и импортов
- Визуальный блочный редактор
- Текстовый редактор кода

**Режимы работы:**
- `preview` — просмотр файла
- `edit` — визуальное редактирование блоков
- `code` — текстовое редактирование

### 2. BlockEditorPanel.jsx

Панель управления блочным редактором:
- Редактирование стилей элементов
- Управление слоями (layers)
- Перемещение и изменение размера элементов
- Вставка/удаление блоков

### 3. FileTree.jsx

Компонент файлового дерева проекта:
- Навигация по файлам
- Открытие файлов
- Создание/удаление файлов и папок
- Переименование

### 4. Block Editor System

Система визуального редактирования:

**Инструментирование:**
- `HtmlInstrumenter` — добавляет `data-mrpak-id` атрибуты к HTML элементам
- `JsxInstrumenter` — добавляет `data-mrpak-id` к JSX элементам в исходном коде

**Patch Engine:**
- Применение изменений стилей
- Вставка/удаление элементов
- Изменение текста
- Перемещение элементов (reparent)

**Протокол сообщений:**
- `MRPAK_MSG` — сообщения от iframe к React (SELECT, APPLY, TREE и т.д.)
- `MRPAK_CMD` — команды от React к iframe (INSERT, DELETE, SET_STYLE и т.д.)

### 5. WebView.jsx

Компонент для отображения HTML контента в iframe:
- Рендеринг HTML/React компонентов
- Обработка сообщений от блочного редактора
- Интеграция с BlockEditorPanel

## Горячие клавиши

### Глобальные клавиши

| Комбинация | Действие | Контекст |
|------------|----------|----------|
| `Ctrl+S` / `Cmd+S` | Сохранить файл | Везде |
| `Ctrl+Z` / `Cmd+Z` | Отменить последнее действие | Режим редактирования (edit) |
| `Ctrl+Shift+Z` / `Cmd+Shift+Z` | Повторить отмененное действие | Режим редактирования (edit) |

### Особенности сохранения

- **В режиме `code`:** Сохраняет текущее содержимое из Monaco Editor
- **В режиме `edit`:** Применяет накопленные изменения (staged patches) к исходному файлу
- **Визуальная обратная связь:** Зеленая галочка "✓ Сохранено" появляется на 2 секунды

### История изменений (Undo/Redo)

Поддерживаемые операции:
- ✏️ **Изменение стилей** — отмена/повтор изменений CSS-свойств
- ➕ **Добавление блоков** — отмена вставки элементов
- ➖ **Удаление блоков** — восстановление удаленных элементов
- 📝 **Изменение текста** — отмена изменений текстового содержимого

**Важно:** История очищается после сохранения файла (Ctrl+S)

### Кнопки в интерфейсе

В режиме редактирования (edit) доступны кнопки:
- **⟲ Отмена** — отменить последнее действие
- **⟳ Повтор** — повторить отмененное действие

Кнопки автоматически деактивируются, когда история пуста.

## Разработка

### Структура модулей

Проект использует модульную архитектуру:

- **features/** — функциональные модули
  - `file-renderer/` — обработка и рендеринг файлов
  - `file-operations/` — операции с файлами

- **shared/** — общие утилиты
  - `api/` — API для работы с Electron
  - `lib/` — вспомогательные библиотеки
  - `ui/` — переиспользуемые UI компоненты

- **blockEditor/** — система блочного редактора
  - Инструментирование кода
  - Движок патчей
  - Протокол сообщений

### Добавление новых функций

1. **Новые IPC методы:**
   - Добавить обработчик в `electron/main.js` через `ipcMain.handle`
   - Добавить метод в `electron/preload.js` через `contextBridge`
   - Создать обертку в `src/shared/api/electron-api.js`

2. **Новые UI компоненты:**
   - Размещать в `src/shared/ui/` для переиспользования
   - Или в `src/features/` для специфичной функциональности

3. **Обработка новых типов файлов:**
   - Добавить детектор в `src/shared/lib/file-type-detector.js`
   - Добавить обработчик в `src/RenderFile.jsx`

### Отладка

- **Main Process:** Используйте `console.log` в `electron/main.js` (вывод в терминал)
- **Renderer Process:** Используйте DevTools (открываются автоматически в dev режиме)
- **IPC:** Проверяйте сообщения в DevTools Console
- **iframe коммуникация:** Логи с префиксами `[iframe ...]` показывают операции внутри iframe
- **История изменений:** Логи с префиксами `⏮️ [Undo]` и `⏩ [Redo]` отслеживают операции истории
- **Блочный редактор:** Логи с префиксами `[BlockEditorPanel]` и `[stageInsertBlock]` показывают операции редактора

### Советы по использованию

**Режимы работы:**

1. **Preview** (Просмотр)
   - Используйте для визуального просмотра файла
   - Изменения невозможны
   - Быстрая загрузка

2. **Edit** (Редактирование)
   - Визуальное редактирование элементов
   - Клик по элементу для выбора
   - Панель справа для изменения свойств
   - Используйте Ctrl+S для сохранения
   - Используйте Ctrl+Z для отмены

3. **Code** (Код)
   - Текстовое редактирование в Monaco Editor
   - Подсветка синтаксиса
   - Автодополнение
   - Ctrl+S для сохранения

**Работа с блоками:**

1. **Выбор элемента:** Кликните на элемент в iframe
2. **Изменение стилей:** Используйте панель справа
3. **Добавление блока:** Выберите режим (child/sibling), введите HTML и нажмите "Добавить"
4. **Удаление блока:** Выберите элемент и нажмите "Удалить"
5. **Сохранение:** Ctrl+S применяет все изменения к файлу

**Лучшие практики:**

- ✅ Часто сохраняйте изменения (Ctrl+S)
- ✅ Используйте preview режим для быстрого просмотра
- ✅ Тестируйте изменения перед сохранением
- ✅ Используйте Undo (Ctrl+Z) для экспериментов
- ⚠️ История очищается после сохранения
- ⚠️ Переключение между файлами сбрасывает несохраненные изменения

**Производительность:**

- Проект оптимизирован для работы с файлами до 10,000 строк
- Большие проекты могут замедлять работу блочного редактора
- Используйте code режим для больших файлов

## Особенности реализации

### Обработка зависимостей

Приложение автоматически:
- Загружает локальные импорты из React/React Native файлов
- Разрешает пути с поддержкой `@/` алиасов
- Обрабатывает относительные пути (`./`, `../`)
- Встраивает зависимости в HTML для рендеринга

### Блочный редактор

- **Идентификация элементов:** Каждый элемент получает уникальный `data-mrpak-id` атрибут
- **Дерево слоев:** Автоматически строится иерархическая структура компонентов
- **Staged Patches:** Изменения накапливаются локально перед применением к файлу
- **Применение изменений:** Автоматически при сохранении (Ctrl+S) или смене режима

### Система Staged Patches

**Как это работает:**

1. **Внесение изменений** — все операции (стили, вставка, удаление) накапливаются в памяти
2. **Визуальный предпросмотр** — изменения сразу видны в iframe, но не сохранены в файл
3. **Индикатор изменений** — показывает наличие несохраненных изменений
4. **Сохранение** — при Ctrl+S все накопленные патчи применяются к исходному коду файла

**Преимущества:**
- ✅ Возможность просмотра результата до сохранения
- ✅ Групповое применение изменений
- ✅ Возможность отмены всех изменений без сохранения
- ✅ Полная поддержка Undo/Redo перед сохранением

**Типы патчей:**
- `patch` — изменение стилей элемента
- `insert` — вставка нового элемента
- `delete` — удаление элемента
- `setText` — изменение текстового содержимого
- `reparent` — перемещение элемента в дереве

### Инструментирование DOM

После рендеринга React компонентов:
- DOM элементы автоматически получают `data-mrpak-id`
- MutationObserver отслеживает новые элементы
- Дерево слоев обновляется автоматически

### Архитектура Undo/Redo

**Структура истории:**

```javascript
// Стек отмены
undoStack = [
  { type: 'patch', blockId: 'id1', patch: {...}, previousValue: {...} },
  { type: 'insert', blockId: 'id2', targetId: 'id3', ... },
  { type: 'delete', blockId: 'id4', ... }
]

// Стек повтора
redoStack = [
  { type: 'patch', blockId: 'id5', ... }
]
```

**Поток операций:**

1. **Новая операция:**
   - Добавляется в `undoStack`
   - `redoStack` очищается

2. **Отмена (Undo):**
   - Операция извлекается из `undoStack`
   - Применяется обратная операция
   - Операция добавляется в `redoStack`

3. **Повтор (Redo):**
   - Операция извлекается из `redoStack`
   - Применяется повторно
   - Операция добавляется в `undoStack`

4. **Сохранение файла:**
   - Оба стека очищаются
   - История начинается заново

**Защита от дублирования:**
- Debounce на уровне UI (300ms для кликов)
- Debounce на уровне операций (500ms для идентичных операций)
- Защита от двойного рендеринга (отключен React.StrictMode в production)

### Коммуникация с iframe

**Протокол MRPAK (Message & Command):**

**От iframe к React (`MRPAK_MSG`):**
```javascript
// Выбор элемента
{ type: 'SELECT', id: 'mrpak-123' }

// Применение изменений
{ type: 'APPLY', id: 'mrpak-123', patch: { color: 'red' } }

// Обновление дерева
{ type: 'TREE', tree: [...] }
```

**От React к iframe (`MRPAK_CMD`):**
```javascript
// Установка стилей
{ type: 'SET_STYLE', id: 'mrpak-123', patch: { color: 'blue' } }

// Вставка элемента
{ type: 'INSERT', targetId: 'mrpak-123', mode: 'child', html: '<div>...</div>' }

// Удаление элемента
{ type: 'DELETE', id: 'mrpak-123' }
```

**Особенности:**
- Все команды логируются для отладки
- Поддержка `null` значений для удаления стилей
- Автоматическое обновление дерева после изменений

## Известные ограничения и решения

### Ограничение 1: Дублирование блоков

**Проблема:** В некоторых случаях при быстром двойном клике могут создаваться дубликаты.

**Решение:** Реализована защита через debounce:
- 300ms на уровне UI
- 500ms на уровне операций
- Отключен React.StrictMode

### Ограничение 2: История после сохранения

**Проблема:** После Ctrl+S история Undo/Redo очищается.

**Объяснение:** Это сделано намеренно, так как изменения уже применены к файлу на диске.

**Решение:** Сохраняйте изменения только когда уверены в результате.

### Ограничение 3: Большие файлы

**Проблема:** Файлы размером > 10,000 строк могут замедлять блочный редактор.

**Решение:** Используйте режим Code для редактирования больших файлов.

### Ограничение 4: Внешние стили

**Проблема:** CSS классы из внешних файлов не отображаются в блочном редакторе.

**Решение:** Используйте inline стили или редактируйте CSS файлы отдельно.

### Ограничение 5: Сложные React компоненты

**Проблема:** Компоненты с условным рендерингом или циклами могут отображаться некорректно.

**Решение:** Используйте режим Code для таких компонентов.
